# 🤝 Contributing

We welcome any contributions to the **DevoMultiX** project!
Whether you are contributing code, improving the documentation, or reporting bugs – all help is appreciated.

## 🚀 How can I get started?

1. **Check Issues**: Search our GitHub repository for open issues (especially those tagged `good first issue`).
2. **Fork**: Create a fork of the repository.
3. **Create Branch**: Create a branch for your changes (`git checkout -b feature/MyNewFeature`).
4. **Commit & Push**: Commit your changes and push them.
5. **Pull Request**: Open a PR and tell us what you have done!

## 📝 Writing Documentation

This documentation is based on **MkDocs** using the **Material for MkDocs** theme.
- Write in Markdown (`.md`).
- Use admonitions (e.g., `!!! info`, `!!! warning`) when appropriate.
- Keep the style modern and clean.
